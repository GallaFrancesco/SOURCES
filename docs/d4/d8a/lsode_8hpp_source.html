339 /*
340 c-----------------------------------------------------------------------
341 c this is the march 30, 1987 version of
342 c lsoda.. livermore solver for ordinary differential equations, with
343 c automatic method switching for stiff and nonstiff problems.
344 c
345 c this version is in double precision.
346 c
347 c lsoda solves the initial value problem for stiff or nonstiff
348 c systems of first order ode-s,
349 c dy/dt = f(t,y) , or, in component form,
350 c dy(i)/dt = f(i) = f(i,t,y(1),y(2),...,y(neq)) (i = 1,...,neq).
351 c
352 c this a variant version of the lsode package.
353 c it switches automatically between stiff and nonstiff methods.
354 c this means that the user does not have to determine whether the
355 c problem is stiff or not, and the solver will automatically choose the
356 c appropriate method. it always starts with the nonstiff method.
357 c
358 c authors..
359 c linda r. petzold and alan c. hindmarsh,
360 c computing and mathematics research division, l-316
361 c lawrence livermore national laboratory
362 c livermore, ca 94550.
363 c
364 c references..
365 c 1. alan c. hindmarsh, odepack, a systematized collection of ode
366 c solvers, in scientific computing, r. s. stepleman et al. (eds.),
367 c north-holland, amsterdam, 1983, pp. 55-64.
368 c 2. linda r. petzold, automatic selection of methods for solving
369 c stiff and nonstiff systems of ordinary differential equations,
370 c siam j. sci. stat. comput. 4 (1983), pp. 136-148.
371 c-----------------------------------------------------------------------
372 c summary of usage.
373 c
374 c communication between the user and the lsoda package, for normal
375 c situations, is summarized here. this summary describes only a subset
376 c of the full set of options available. see the full description for
377 c details, including alternative treatment of the jacobian matrix,
378 c optional inputs and outputs, nonstandard options, and
379 c instructions for special situations. see also the example
380 c problem (with program and output) following this summary.
381 c
382 c a. first provide a subroutine of the form..
383 c subroutine f (neq, t, y, ydot)
384 c dimension y(neq), ydot(neq)
385 c which supplies the vector function f by loading ydot(i) with f(i).
386 c
387 c b. write a main program which calls subroutine lsoda once for
388 c each point at which answers are desired. this should also provide
389 c for possible use of logical unit 6 for output of error messages
390 c by lsoda. on the first call to lsoda, supply arguments as follows..
391 c f = name of subroutine for right-hand side vector f.
392 c this name must be declared external in calling program.
393 c neq = number of first order ode-s.
394 c y = array of initial values, of length neq.
395 c t = the initial value of the independent variable.
396 c tout = first point where output is desired (.ne. t).
397 c itol = 1 or 2 according as atol (below) is a scalar or array.
398 c rtol = relative tolerance parameter (scalar).
399 c atol = absolute tolerance parameter (scalar or array).
400 c the estimated local error in y(i) will be controlled so as
401 c to be less than
402 c ewt(i) = rtol*abs(y(i)) + atol if itol = 1, or
403 c ewt(i) = rtol*abs(y(i)) + atol(i) if itol = 2.
404 c thus the local error test passes if, in each component,
405 c either the absolute error is less than atol (or atol(i)),
406 c or the relative error is less than rtol.
407 c use rtol = 0.0 for pure absolute error control, and
408 c use atol = 0.0 (or atol(i) = 0.0) for pure relative error
409 c control. caution.. actual (global) errors may exceed these
410 c local tolerances, so choose them conservatively.
411 c itask = 1 for normal computation of output values of y at t = tout.
412 c istate = integer flag (input and output). set istate = 1.
413 c iopt = 0 to indicate no optional inputs used.
414 c rwork = real work array of length at least..
415 c 22 + neq * max(16, neq + 9).
416 c see also paragraph e below.
417 c lrw = declared length of rwork (in user-s dimension).
418 c iwork = integer work array of length at least 20 + neq.
419 c liw = declared length of iwork (in user-s dimension).
420 c jac = name of subroutine for jacobian matrix.
421 c use a dummy name. see also paragraph e below.
422 c jt = jacobian type indicator. set jt = 2.
423 c see also paragraph e below.
424 c note that the main program must declare arrays y, rwork, iwork,
425 c and possibly atol.
426 c
427 c c. the output from the first call (or any call) is..
428 c y = array of computed values of y(t) vector.
429 c t = corresponding value of independent variable (normally tout).
430 c istate = 2 if lsoda was successful, negative otherwise.
431 c -1 means excess work done on this call (perhaps wrong jt).
432 c -2 means excess accuracy requested (tolerances too small).
433 c -3 means illegal input detected (see printed message).
434 c -4 means repeated error test failures (check all inputs).
435 c -5 means repeated convergence failures (perhaps bad jacobian
436 c supplied or wrong choice of jt or tolerances).
437 c -6 means error weight became zero during problem. (solution
438 c component i vanished, and atol or atol(i) = 0.)
439 c -7 means work space insufficient to finish (see messages).
440 c
441 c d. to continue the integration after a successful return, simply
442 c reset tout and call lsoda again. no other parameters need be reset.
443 c
444 c e. note.. if and when lsoda regards the problem as stiff, and
445 c switches methods accordingly, it must make use of the neq by neq
446 c jacobian matrix, j = df/dy. for the sake of simplicity, the
447 c inputs to lsoda recommended in paragraph b above cause lsoda to
448 c treat j as a full matrix, and to approximate it internally by
449 c difference quotients. alternatively, j can be treated as a band
450 c matrix (with great potential reduction in the size of the rwork
451 c array). also, in either the full or banded case, the user can supply
452 c j in closed form, with a routine whose name is passed as the jac
453 c argument. these alternatives are described in the paragraphs on
454 c rwork, jac, and jt in the full description of the call sequence below.
455 c
456 c-----------------------------------------------------------------------
457 */

529 void intdy(double t, int k, double *dky, int *iflag);
530 /*
531  Intdy computes interpolated values of the k-th derivative of the
532  dependent variable vector y, and stores it in dky. This routine
533  is called within the package with k = 0 and *t = tout, but may
534  also be called by the user for any k up to the current order.
535  ( See detailed instructions in the usage documentation. )
536 
537  The computed values in dky are gotten by interpolation using the
538  Nordsieck history array yh. This array corresponds uniquely to a
539  vector-valued polynomial of degree nqcur or less, and dky is set
540  to the k-th derivative of this polynomial at t.
541  The formula for dky is
542 
543  q
544  dky[i] = sum c[k][j] * ( t - tn )^(j-k) * h^(-j) * yh[j+1][i]
545  j=k
546 
547  where c[k][j] = j*(j-1)*...*(j-k+1), q = nqcur, tn = tcur, h = hcur.
548  The quantities nq = nqcur, l = nq+1, n = neq, tn, and h are declared
549  static globally. The above sum is done in reverse order.
550  *iflag is returned negative if either k or t is out of bounds.
551 */

553 void cfode(int meth);
554 /*
555  cfode is called by the integrator routine to set coefficients
556  needed there. The coefficients for the current method, as
557  given by the value of meth, are set for all orders and saved.
558  The maximum order assumed here is 12 if meth = 1 and 5 if meth = 2.
559  ( A smaller value of the maximum order is also allowed. )
560  cfode is called once at the beginning of the problem, and
561  is not called again unless and until meth is changed.
562 
563  The elco array contains the basic method coefficients.
564  The coefficients el[i], 1 < i < nq+1, for the method of
565  order nq are stored in elco[nq][i]. They are given by a generating
566  polynomial, i.e.,
567 
568  l(x) = el[1] + el[2]*x + ... + el[nq+1]*x^nq.
569 
570  For the implicit Adams method, l(x) is given by
571 
572  dl/dx = (x+1)*(x+2)*...*(x+nq-1)/factorial(nq-1), l(-1) = 0.
573 
574  For the bdf methods, l(x) is given by
575 
576  l(x) = (x+1)*(x+2)*...*(x+nq)/k,
577 
578  where k = factorial(nq)*(1+1/2+...+1/nq).
579 
580  The tesco array contains test constants used for the
581  local error test and the selection of step size and/or order.
582  At order nq, tesco[nq][k] is used for the selection of step
583  size at order nq-1 if k = 1, at order nq if k = 2, and at order
584  nq+1 if k = 3.
585 */

589 void prja(int neq, double *y, class SystEq& f, void *_data);
590 /*
591  prja is called by stoda to compute and process the matrix
592  P = I - h * el[1] * J, where J is an approximation to the Jacobian.
593  Here J is computed by finite differencing.
594  J, scaled by -h * el[1], is stored in wm. Then the norm of J ( the
595  matrix norm consistent with the weighted max-norm on vectors given
596  by vmnorm ) is computed, and J is overwritten by P. P is then
597  subjected to LU decomposition in preparation for later solution
598  of linear systems with p as coefficient matrix. This is done
599  by dgefa if miter = 2, and by dgbfa if miter = 5.
600 */
